# 日序 · 项目规则

产品名：**日序**（日 / 周计划本地工具）。  
主文件：`daily-plan-template.html`。不是个人档案表单、不是演示站。

必读：

- `docs/对话沉淀-需求与教训.md` — 需求演进、踩坑、用户原话映射  
- `.grok/skills/make-plan-product/SKILL.md`  
- `.grok/skills/make-plan-ui/SKILL.md`  
- `.grok/skills/make-plan-lessons/SKILL.md`  

---

## 产品硬约束

1. **用途是制定计划**，禁止姓名、身份、角色等档案字段。  
2. **默认空白**。除用户点「示例 / 填入示例」外，禁止向表单写 mock。  
3. **示例**须同时填满日+周；禁止产品自述文案。  
4. **清空** = 清空**当前查看日**（日期不变），完整保留周计划，立即持久化；先 `cancelScheduledSave()`；**不误清其它日期**。  
5. **日闭环诚实**：  
   - 存档：`daily-plan-day-v2:{date}`、`daily-plan-week-v2:{monday}`  
   - **`→ 明日` / 滚入**：只写 `review.tomorrowMit`，**不改日期**  
   - **开启新的一天**：日期+1，明日 MIT→新日 MIT，清块/复盘/睡觉仪式等  
6. **MIT**：默认 3，最少 3，最多 6；#1=今日一件大事。  
7. **日期 / 星期 / 周**：  
   - 星期顺序周一→周日  
   - 改日期、改星期 → **`switchToDate`**，不灌周主题 mock  
   - 上/下周、定位本周 → **`switchToWeek`**  
   - **`convergeWeekFocus` 与表单 weekFocus 互写：仅当前日 ∈ 打开周**（存盘+输入都要门闩）  
   - 跨周且新周无档：空 `days`，禁止克隆上周主题日  
   - 预览「今天主题」仅日∈打开周  
8. **审查**：三态 ok / warn / skip；skip≠通过；滚入须未完成**全部**出现在明日清单。  
9. **必做 / 必回应**：  
   - 晨间 ≥1 勾选  
   - 运动与休息：必选是否运动；不做要理由  
   - 晚间复盘：必选是否复盘；不复盘要理由  
10. **睡觉仪式**：多项任选（冥想/正念/写日记/拉伸等）+ 睡觉时间 +「可以安心睡了」（**独立确认**，不强制全勾联动）；写日记可链到心境日记 `https://xinjing.cyforkk.top/`。  
11. **事项字段用条目列表**（一条一项）：运动安排、边界四块、反思三问、感恩、明天、周成功/回顾；预览与 MD 同构。理由类用 textarea。  
12. **文案白话**：专注工作/杂事琐事/运动与休息；禁止难词当主标签。  
13. **预览与导出**走 **`buildPlanView`**，禁止三套各写各的。  
14. **Git 提交中文**。Windows 提交勿用 bash heredoc。

## UI 硬约束

1. **去 AI 感**：禁止 indigo/紫渐变、标题渐变字、背景光晕、强玻璃拟态、大胶囊重阴影、hover 抬升。纸色底 + 墨色主按钮 + 小圆角细边框。  
2. **主题按钮**：跟随系统 / 浅 / 深，写入 `localStorage`。  
3. **自定义下拉**：选项纯文字；触发器保留箭头；fixed 菜单在页面滚动/resize 时关闭（菜单内滚动除外）。  
4. **手机**：单栏；表内横滑不撑破整页；主操作可底栏；桌面勿重复堆固定底栏。  
5. **体验**：主路径短；低频区折叠；同一动作不要多处入口；条目列表交互统一。

## 实现约定

- 主文件：`daily-plan-template.html`  
- 改字段同步：normalize / 表单 / `buildPlanView` 消费方 / 审查 / 示例 / 清空 / docs / skill（若行为变）

- 明确不做：完整四象限 UI、番茄钟、云账号、AI 排程、名人作息默认表
