# make_plan 教训技能

完整对照：`docs/对话沉淀-需求与教训.md`。

修 bug 前：先在下表搜是否旧坑；是则按「正确做法」修，不要另开旁路。

## 高危旧坑（近期仍易复发）

| 坑 | 症状 | 正确修法 |
|---|---|---|
| weekFocus **仅存盘门闩** | 日在 A 周、面板在 B 周，改输入框互污染 | `bindFormEvents` 也要 `dayBelongsToOpenWeek()` |
| 清空强制今天 | 看明天点清空 → 今天被清空、明天还在 | **清空当前查看日**，日期不变 |
| 防抖写回 | 切日/清空后旧 timer 写脏档 | `cancelScheduledSave()` |
| weekStrip 张冠李戴 | 日∉周仍显示邻周「今天主题」 | 仅日∈打开周才取 `week.days` |
| 预览有、MD 无 | 主题条/列表结构不一致 | 只消费 `buildPlanView`，MD/TXT 同步段 |
| 表单互写总确认×多仪式 | 睡觉仪式 12 项全勾才确认 | 「可以安心睡了」**独立** |
| 事项用大 textarea | 用户要一条一个，MD 也一条条 | 公共条目列表 |
| 难词 / 堆勾选 | 「易懂」失败；仪式像问卷 | 白话；任选不必堆满 |
| 清空/开新日漏字段 | 新字段清不掉或示例不全 | 过字段同步清单 |

## 回归清单（改 HTML 后尽量点）

### A. Mock / 清空

1. 清除全部 → 刷新全空  
2. 仅「示例」出演示；日+周完整；无产品自述  
3. **清空当前查看日** → 该日空、日期不变、周在  
4. 查看**非今天** → 清空 → **今天档完好**，清的是查看日  
5. 改星期 → `switchToDate` 加载该日；不灌主题  

### B. 日闭环

1. `→ 明日`：明日清单更新，**日期不变**  
2. 开启新的一天：日期+1，MIT 承接，复盘/睡觉仪式清空  
3. 次日有档：可选合并 MIT  
4. 跨周开新日（新周无档）：主题日**空**  

### C. 周与 focus

1. 写本周重点 → 下一周 focus 不脏 → 回本周仍在  
2. 日停 A 周、面板翻 B 周 → **改任一侧输入框不互写**  
3. 保存后邻周档 focus 仍空/旧值正确  

### D. 日期星期

1. 改日期 → 星期正确 + 切档  
2. 改星期 → 同周对应日 + 切档  
3. 顺序：周一…周日  

### E. 必做模块 / 审查 / 导出

1. 晨间未勾 → warn  
2. 运动未选 yes/no → warn；no 无理由 → warn  
3. 复盘未选 → warn；no 无理由 → warn  
4. 睡觉仪式：勾几项或设睡觉时间或「可以安心睡了」→ ok  
5. 部分滚入 → warn；全覆盖 → ok  
6. 审查非全绿冒充；有 skip/warn  
7. 预览与 MD/TXT：**列表一条一项**、主题条一致  

### F. 条目列表

1. +一项 / ×删除 / Enter 插行  
2. 粘贴多行拆成多条  
3. 旧换行文本存档能拆开显示  
4. 预览与导出结构一致  

### G. UI

1. 下拉开后滚页面即关  
2. 手机示例无整页左右滑  
3. 无 AI 紫渐变；主题按钮可用  
4. 文案无「深工」等黑话  

## 修 bug 顺序

1. 用一句复述用户期望（对照「用户原话」表）  
2. 对照本 skill + 对话沉淀是否旧坑  
3. 改代码 + 过字段同步清单（`buildPlanView`/审查/示例/清空）  
4. 跑本清单相关项  
5. 更新 `docs/功能-*.md` 或 bug 记录  
6. 若行为规则变了 → **同步 product skill 与 prompts**  
7. 中文 commit（若用户要求）  

## 原则

- 不要用新功能掩盖旧坑。  
- 日切不诚实、mock 泄漏、串档、清空误伤 → **先修闭环与边界**。  
- 用户说「a/b 方案」→ 按确认方案改，并写进沉淀，避免下轮又改回。
