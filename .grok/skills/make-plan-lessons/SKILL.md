# make_plan 教训技能

完整对照：`docs/对话沉淀-需求与教训.md`。

## 回归清单（改 HTML 后尽量点）

### A. Mock / 清空

1. 清除全部 → 刷新全空  
2. 仅「示例」出演示；日+周完整；无产品自述  
3. 清空本日 → 日空、周在  
4. 改星期 → `switchToDate` 加载该日；不灌主题  

### B. 日闭环

1. `→ 明日`：明日清单更新，**日期不变**  
2. 开启新的一天：日期+1，MIT 承接，复盘清空  
3. 次日有档：可选合并 MIT  
4. 跨周开新日（新周无档）：主题日**空**  

### C. 周与 focus

1. 写本周重点 → 下一周 focus 不脏 → 回本周仍在  
2. 日停在 A 周、面板翻 B 周 → 保存不把 A 的重点写入 B  

### D. 日期星期

1. 改日期 → 星期正确 + 切档  
2. 改星期 → 同周对应日 + 切档  
3. 顺序：周一…周日  

### E. 审查 / 关机 / 导出

1. 非全绿；有 skip/warn  
2. 部分滚入 → warn；全覆盖 → ok  
3. 关机双向  
4. 预览与导出 MD 字段一致（`buildPlanView`）  

### F. UI

1. 下拉开后滚页面即关  
2. 手机示例无整页左右滑  
3. 无 AI 紫渐变；主题按钮可用  

## 用户原话速查

见 `docs/对话沉淀-需求与教训.md` 第三节。

## 修 bug 顺序

1. 复述期望一句  
2. 对照教训表是否旧坑  
3. 改代码 + `buildPlanView`/审查/示例若涉及  
4. 跑本清单相关项  
5. 中文 commit（若用户要求）  

## 原则

不要用新功能掩盖旧坑。日切不诚实或 mock 泄漏时，先修闭环与边界。
