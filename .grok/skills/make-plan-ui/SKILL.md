# make_plan UI 技能

## 先读

- `AGENTS.md` UI 硬约束  
- `docs/对话沉淀-需求与教训.md` UI 表  
- `docs/UX精简说明.md`  
- 业务控件形态见 `make-plan-product`（条目列表 / 必做）  

## 视觉：去 AI 化（强制）

**禁止**：紫/indigo 主色与渐变按钮、标题渐变字、radial 光晕、强 blur 玻璃、999px 胶囊+重阴影、卡片 hover 抬升、分区彩色装饰点。

**推荐**：纸色底、墨色字与主按钮、4–6px 圆角、细边框、少阴影；夜间中性深灰无霓虹。打印强制浅色。

主题：**按钮**循环 跟随系统 → 浅 → 深，键如 `daily-plan-theme-v1`。

## 文案：白话优先

用户明确要求「整个使用要易懂」。

| 避免 | 使用 |
|---|---|
| 深工 / 浅工 / Deep Work 黑话作主标签 | 专注工作 / 杂事琐事 |
| 身体与恢复 | 运动与休息 |
| 关机仪式 / 收工与睡觉（堆满勾选） | 睡觉仪式（任选几项） |
| 固定睡前时间 | 睡觉时间 |
| 长教学占模块首行 | 一句 hint + 底部方法参考 |
| 产品自述 | 中性计划句 |

必做模块用短 `req-tag`（必须 / 必须回应），勿大段恐吓文案。

## 自定义下拉（cselect）

1. 选项纯文字（可有灰色说明）；无色点/勾选图标。  
2. 触发器保留 chevron。  
3. 打开时 fixed 贴触发器。  
4. 页面/容器滚动关闭；菜单内滚动不关。  
5. resize 关闭；关闭时清 inline position。  
6. 星期顺序：周一…周日。  

## 条目列表交互

事项字段统一「一行一项」观感：

- 标签「安排 1 / 安排 2…」或等价  
- `+ 一项`；多于最少条时 `×` 删除  
- Enter 在下方插空行；粘贴多行自动拆条  
- 与预览/MD 列表同构  

实现复用 `renderItemList`，禁止每个字段复制一套 DOM 逻辑。  
**理由类**（不运动/不复盘）仍用 textarea。

## 布局与手机

- 默认单栏；宽屏可双栏。  
- `minmax(0,1fr)`；宽表 overflow-x 表内滑。  
- **粘性主操作仅手机**；桌面靠顶栏，避免双份噪音。  
- 日/周不对齐时可用 `align-banner` 提示。  
- 睡觉仪式等多项勾选：两列 grid，避免一列过长。  

## 体验：砍多余

主路径：概览 → MIT → 时间块 → 护栏 → 晨间/运动 → 复盘与睡觉 → 开启新的一天。  

| 倾向删除/合并 | 保留 |
|---|---|
| 大 Hero/四步流程卡占首屏 | 短标题或无 |
| 同动作多按钮（→明日×3） | 一处主入口+手机底栏 |
| 每区长教学 hint | 底部方法参考 |
| 起身提醒等「提醒自己」噪音 | 去掉 |
| 顶栏过多锚点 | 审查/参考即可 |

按钮分层：主（开新日）/ 次（保存、示例）/ 危险沉底。
