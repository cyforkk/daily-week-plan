# make_plan 产品技能

## 先读

- `AGENTS.md`
- `docs/对话沉淀-需求与教训.md`
- 必要时：`docs/产品业务与代码审查.md`

## 定位

产品名 **日序**。用户打开是为了**写今天/本周的计划**，不是填档案，不是看产品广告示例。

## 必守规则

### 1. 无档案字段

禁止姓名、身份、角色。概览仅：日期、星期、能量、主题、意图等计划项。

### 2. Mock 边界

| 允许 | 禁止 |
|---|---|
| 用户点「示例」写入完整演示 | 改日期/星期、加载、切周时自动填长文案 |
| 底部方法参考/技巧（教学区） | 产品自述示例句 |

示例必须**日+周**都完整。

### 3. 清空

- **清空本日**：只清日；保留 `week`；立即写盘。  
- **清除全部**：日+周+索引（危险确认）。

### 4. 日闭环语义（勿混淆）

| 操作 | 含义 |
|---|---|
| `→ 明日` / 滚入 | 未完成 MIT → `tomorrowMit`（去重≤6）；**不改日期** |
| 开启新的一天 | 存今日 → 日期+1 → 明日 MIT 成新 MIT；清完成/块/复盘/关机；保留年季与周 |
| 次日已有档 | 加载存档；可选**合并**今日明日 MIT；同文案完成态尽量保留 |

### 5. 存储与切换

- 日：`daily-plan-day-v2:{YYYY-MM-DD}`  
- 周：`daily-plan-week-v2:{周一}`  
- 改日期 / 改星期 → **`switchToDate`**  
- 上/下周、本周、改周一 → **`switchToWeek`**  
- **`convergeWeekFocus` 仅 `dayBelongsToOpenWeek()`**  
- 跨周新周无档：空 `days`，不 `deepClone` 上周  

### 6. MIT

默认 3，最少 3，最多 6；#1 今日一件大事；支持完成勾选。

### 7. 审查

- 三态：ok / warn / skip；skip≠通过  
- 日终项未复盘前可为 skip  
- 滚入：未完成须**全部**出现在明日清单才 ok  
- 文案按钮名用「→ 明日」  

### 8. 关机

四步与总确认双向。

### 9. 字段变更同步清单

1. state / `normalizeState`  
2. 表单  
3. **`buildPlanView`**（预览与导出只消费它）  
4. `renderPreview` / `toMarkdown` / `toPlainText` 若结构变化  
5. `auditPlan`  
6. 示例 `sampleState`  
7. 清空逻辑  
8. docs（若对外行为变）  

### 10. 扩展方法时

抽象模块，不抄作息钟点。优先：对齐、MIT、时间块、护栏、通信窗、复盘、关机、周主题。  
不做：完整四象限、番茄钟、云账号、AI 排程。

### 11. 提交

中文 commit；Windows 避免 bash heredoc。

## 验收口令

- [ ] 冷启动空白  
- [ ] 示例完整无自述；清空本日留周  
- [ ] 改星期/翻周不串档不污 focus  
- [ ] 滚入≠换日；开新日=换日+MIT  
- [ ] 预览与 MD 一致（`buildPlanView`）
