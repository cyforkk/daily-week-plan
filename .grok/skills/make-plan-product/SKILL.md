# make_plan 产品技能

## 先读

1. `AGENTS.md`  
2. `docs/对话沉淀-需求与教训.md`（需求演进 + 踩坑表）  
3. 改相关模块时：`docs/功能-*.md`、`docs/审查-*.md`  
4. 教训回归：`make-plan-lessons`  

## 定位

产品名 **日序**。用户打开是为了**写今天/本周的计划**。

- 不是档案表（无姓名/身份）  
- 不是营销演示站（默认空白，仅「示例」灌 mock）  
- 不是抄名人作息表  
- **文案白话**，黑话会伤害「易懂」  

## 必守规则

### 1. 无档案字段

概览仅：日期、星期、主题、意图等计划项。禁止姓名、身份、角色。

### 2. Mock 边界

| 允许 | 禁止 |
|---|---|
| 用户点「示例」写入完整演示 | 改日期/星期、加载、切周时自动填长文案 |
| 底部方法参考/技巧（教学区） | 产品自述示例句 |

示例必须**日+周**都完整、中性文案。

### 3. 清空（方案 A）

- **清空** = 清空 **`state.meta.date` 当前查看日**，**日期不变**  
- 完整保留 `week`；立即写盘  
- 先 **`cancelScheduledSave()`**，避免 debounce 写回旧内容  
- **禁止**为「回到今天」而覆盖今天档、却不清当前查看日  
- **清除全部**：日+周+索引（危险确认）

### 4. 日闭环语义（勿混淆）

| 操作 | 含义 |
|---|---|
| `→ 明日` / 滚入 | 未完成 MIT → `tomorrowMit`（去重≤6）；**不改日期** |
| 开启新的一天 | 存今日 → 日期+1 → 明日 MIT 成新 MIT；清完成/块/复盘/睡觉仪式；保留年季与周 |
| 次日已有档 | 加载存档；可选**合并**今日明日 MIT；同文案完成态尽量保留 |

### 5. 存储与切换

- 日：`daily-plan-day-v2:{YYYY-MM-DD}`  
- 周：`daily-plan-week-v2:{周一}`  
- 改日期 / 改星期 → **`switchToDate`**（先 cancel 再存再载）  
- 上/下周、本周、改周一 → **`switchToWeek`**  
- **`convergeWeekFocus` 与表单 weekFocus 互写：仅 `dayBelongsToOpenWeek()`**  
  - 存盘有门闩不够，**`bindFormEvents` 里 goals.weekFocus ↔ week.focus 也必须门闩**（P0 旧坑）  
- 跨周新周无档：空 `days`，不 `deepClone` 上周  
- 预览 weekStrip（今天主题/焦点）：**仅日∈打开周**，否则张冠李戴  

### 6. MIT + 原子行动

默认 3，最少 3，最多 6；#1 今日一件大事；支持完成勾选。  
每件可拆 **行动步骤** `goals.mitSteps[i] = [{ text, done }]`（最多 12 步）：+ 一步 / Enter 插行 / 勾完成 / × 删。  
预览与 MD/TXT 须缩进列出步骤。滚入明天、开新日只带要事标题，不强制带步骤。

### 7. 审查

- 三态：ok / warn / skip；**skip≠通过**  
- 日终项未复盘前可为 skip  
- 滚入：未完成须**全部**出现在明日清单才 ok  
- 文案按钮名用「→ 明日」  

### 8. 必做 / 必回应模块

| 模块 | 规则 |
|---|---|
| 晨间仪式 | **必须**；审查 ≥1 勾选 |
| 运动与休息 | **必须回应** yes/no；no → 理由；yes → 类型/安排/时间块等有一项即可 |
| 晚间复盘 | **必须回应** yes/no；no → 理由；yes → 反思/感恩/明天等 |
| 睡觉仪式 | 任选 ≥1 项 **或** 睡觉时间 **或**「可以安心睡了」即可；**不强制全勾** |

### 9. 睡觉仪式（原收工/关机）

可选项见 `SLEEP_RITUAL`：冥想、正念、感恩、自我鼓励、拉伸、深呼吸、少看屏幕、阅读、放松身体、轻音乐、**写日记**、可以放下工作了、杂事已安顿。  
写日记外链：心境日记 `https://xinjing.cyforkk.top/`（新标签打开；顶栏「日记」与睡觉仪式区均有入口）。  

- 另：`review.bedtime`（睡觉时间）、`review.shutdown`（可以安心睡了）  
- **总确认独立**，不要再做「全勾才联动」  
- 旧 `windDown` 读入时迁入对应 shutSteps  

### 10. 条目列表（事项用列表）

**事项**（可数、MD 要 `- 项`）→ 一条一项：

- 运动具体安排 `movementPlans`  
- 坚决不做 / 以后再做 / 请别人做 / 临时想法  
- 进步 / 做得好 / 可改进 / 感恩 / 明天 Top 3  
- 周成功标准 / 周回顾  

**段落**（叙述理由）→ textarea：不运动理由、不复盘理由。  

公共：`asItemList` / `renderItemList` / `getItemListDefs`；支持 +一项、×删、Enter 插行、粘贴多行拆条。  
旧字符串存档：`linesOf` / `asItemList` 兼容。

### 11. 文案白话

| 避免 | 使用 |
|---|---|
| 深工 / 浅工 | 专注工作 / 杂事琐事 |
| 身体与恢复 | 运动与休息 |
| 关机仪式大全 | 睡觉仪式（任选） |
| 固定睡前时间 | 睡觉时间 |
| 产品自述长句 | 中性计划句 |

时间块类型：`专注工作 / 杂事琐事 / 开会 / 回消息 / 运动休息 / 生活`。

### 12. 预览与导出

- **只**经 `buildPlanView`  
- 列表字段预览/MD/TXT 必须**一条一项**（反思、安排、边界等）  
- weekStrip 与预览一致时再导出；落款用**计划日**非系统今天  
- 导出 MD/TXT：浏览器默认下载（`a.download`）；**不做**选择文件夹 / 指定路径  
- 原子行动说明见 `docs/功能-原子行动与下载目录.md`

### 13. 字段变更同步清单

1. `emptyState` / `sampleState` / `normalizeState`  
2. 表单 HTML + 事件  
3. **`buildPlanView`**  
4. `renderPreview` / `toMarkdown` / `toPlainText`  
5. `auditPlan`  
6. 清空 / 开新日重置范围  
7. docs；行为变了更新本 skill / lessons  

### 14. 扩展方法时

抽象模块，不抄作息钟点。优先：对齐、MIT、时间块、护栏、复盘、睡觉仪式、周主题。  
**不做**：完整四象限、番茄钟、云账号、AI 排程、名人默认作息表。

### 15. 提交

中文 commit；Windows 避免 bash heredoc。
