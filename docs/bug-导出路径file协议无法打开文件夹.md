# Bug：导出选文件夹报「无法打开此文件夹…」（已下线该功能）

## 结论

「选择导出文件夹 / 指定路径」功能**已移除**。导出仅走浏览器默认下载。

## 历史现象

选择导出目录时出现：`无法打开此文件夹，因为其中含有系统文件`（常伴随 `file:///`）。

## 历史原因

1. 本地 `file://` 打开页面时，系统目录选择器易失败  
2. `startIn` 指向失效/系统位置  
3. 用户把界面路径当成可打开的 `file:///` 地址  

## 处理

- 去掉目录选择 UI 与 File System Access / IndexedDB 相关代码  
- MD / TXT 恢复为 `a.download` 默认下载
