## 1. 已落地的修复（相对初审）

| 问题 | 处理 |
|---|---|
| 滚入明日不等于换日 | **「开启新的一天」**：保存今日 → 日期+1 → 明日 MIT 写入新 MIT → 清空复盘/块/完成态 |
| 单 key 存档 | **按日分档** `daily-plan-day-v2:{date}` + **按周** `daily-plan-week-v2:{weekStart}` + 索引；兼容 v1 迁移 |
| 改日期不换档 | 改日期 / 改星期导致换日 → `switchToDate` 先存后载 |
| 审查绿勾冒充可选 | **三态** `ok` / `warn` / `skip`；日终项未复盘前为 skip |
| 关机单向 | 总确认 ↔ 四步**全联动**（勾总=全勾，取消总=全清，改任一步重算总） |
| 周 focus 双字段 | **自动双向同步**；去掉手动同步按钮 |
| 周「用作今日」 silently 改日期 | **确认后** `switchToDate` 再写主题 |
| 清空不写盘 | **清空本日**立即保存空日档 |

---

## 2. 当前业务主流程

```
（可选）填周主题/成功标准
  → 日对齐 MIT（#1=一件大事）
  → 时间块 / 护栏
  → 执行勾选完成
  → 复盘 + 滚入明日清单
  → 关机四步
  →【开启新的一天】真正换日并带入 MIT
```

存储：

- 日：`daily-plan-day-v2:YYYY-MM-DD`
- 周：`daily-plan-week-v2:周一日期`
- 索引：`daily-plan-index-v2`
- 兼容备份：`daily-plan-template-v1`

---

## 3. 审查三态含义

| 状态 | 含义 |
|---|---|
| **通过 ok** | 建议达到，已满足 |
| **待改进 warn** | 建议补上 |
| **跳过 skip** | 当前阶段可不做（≠通过） |

日终相关（明日 MIT、关机）仅在「已有复盘/关机迹象」时严格检查。
