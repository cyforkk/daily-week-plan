# 对话沉淀：需求、错误与解法

从本仓库全段协作对话提炼，供 `AGENTS.md`、提示词与 skill 引用。改产品前先对照，避免重复踩坑。

---

## 一、用户需求演进（按对话顺序）

| 阶段 | 用户意图 | 落地要点 |
|---|---|---|
| 起点 | 参考成功人士计划 → 通用模板 → HTML 可输入生成 | 单文件工具，非静态说明 |
| 定位纠正 | **不是填档案**，是制定计划 | 去掉姓名/身份 |
| 控件 | 星期用下拉；顺序周一→周日 | 自定义 cselect |
| MIT | 想起 6 件（Ivy Lee）；后改为默认 3 可加到 6 | `mitCount` 3–6 |
| 研究迭代 | 继续找方法并审查完善 | 关机、延后/委派、审查、周主题… |
| 闭环 | 滚入明日 + 周计划「全部都做」 | 两功能都要，**语义要诚实** |
| 绑定 | 日期星期互相绑定 | 后改为换日=切存档 |
| Mock | 只要「填入示例」；去掉其它自动 mock | 改星期/骨架/预设不得灌长文 |
| 示例文案 | 禁止产品自述句 | 中性演示 |
| 清空 | 清空本日不清周计划示例 | 日/周作用域分离 |
| UI | 现代化 → 去 AI 化；主题要按钮 | 纸质中性 + 三态主题 |
| 下拉 | 选项只要文字但触发器要箭头；滚动错位 | 纯文字列表 + 滚动关闭 |
| 体验 | 按钮布局从使用出发，砍多余设计 | 主路径短、可选折叠 |
| 易懂 | 整个使用要易懂；能力选择不要难词 | 深工→专注工作等；去掉能量/过细 weekday 文案 |
| 晨间 | 晨间仪式必须做 | 审查 ≥1 勾选 |
| 运动 | 身体与恢复 → **运动与休息**；必须回应是否运动；不做要理由；具体安排可一条一条加；去掉「起身提醒」 | `willMove` + `movementPlans[]` |
| 复盘 | 晚间必须回应；不复盘要理由；反思/感恩/明天一条一项 | `willReview` + 条目列表 |
| 收工演进 | 关机四步 → 轻量收工 → **睡觉仪式**（冥想/正念/拉伸等任选） | `SLEEP_RITUAL` + 睡觉时间 +「可以安心睡了」独立 |
| 条目列表 | 很多输入框要「一条一个」，MD 也一条一条 | 公共 `itemList`；事项用列表、感受用段落（后反思也改成列表） |
| 清空语义 | 曾要「清空后日期=今天」；审查后选 **A：清空当前查看日** | 日期不变，不误清其它天 |
| 审查 | 反复审查业务/代码并修 | P0 weekFocus 表单门闩 + 清空 A + P1 |
| 沉淀 | 需求/错误/解法 → 提示词与 skill | 本文件 + `.grok/skills` + `prompts/` |

### 产品一句话

产品名 **日序**：每天打开写日/周计划的本机工具，不是档案表、不是营销演示站、不是抄名人作息表。**文案白话、必做模块要回应、预览与 MD 同构。**

### 已定能力清单（当前）

| 区 | 能力 |
|---|---|
| 日概览 | 日期/星期绑定、主题、意图（无档案） |
| MIT | 默认 3、最多 6、完成勾选、`→ 明日`、开启新的一天 |
| 时间块 | 类型白话：专注工作/杂事琐事/开会/回消息/运动休息/生活 |
| 护栏 | 坚决不做 / 以后再做 / 请别人做 / 临时想法 → **条目列表** |
| 晨间 | **必做**（≥1 勾选）；含淋浴等 |
| 运动与休息 | **必回应** yes/no；no 要理由；yes 可写类型 + **具体安排一条一项** |
| 晚间复盘 | **必回应**；反思/感恩/明天 → 条目列表；预览/MD 一条一项 |
| 睡觉仪式 | 多项任选 + 睡觉时间 +「可以安心睡了」 |
| 周 | 主题日、成功/回顾条目列表、本周重点（日∈周才互写） |
| 存档 | day-v2 / week-v2；审查三态；导出 `buildPlanView` |

## 二、助手犯过的错 → 正确做法

### A. 产品 / 业务

| # | 错误 | 正确做法 |
|---|---|---|
| 1 | 做了姓名、身份档案字段 | 只保留计划字段 |
| 2 | MIT 只 3 或写死 6 格 | 默认 3，可增至 6，可减至 3 |
| 3 | 「滚入明日」= 只写文本，却让用户以为换了日 | **滚入**只写 `tomorrowMit`；**开启新的一天**才日期+1 并承接 MIT |
| 4 | 单一 localStorage key | 按日 / 按周分档 + 索引；改日期 `switchToDate` |
| 5 | 审查「可选」也显示通过 | 三态：ok / warn / **skip≠通过** |
| 6 | 关机总确认与四步单向 | 曾双向；**睡觉仪式多项后总确认改独立**，不强制全勾联动 |
| 7 | 周「用作今日」改日期不换档 | 确认后 `switchToDate` 再写主题 |
| 8 | 改星期灌周主题 mock | 不灌 mock |
| 9 | 改星期只改 label + save 到新日期，覆盖错日 | 改星期也走 **`switchToDate`** |
| 10 | 上/下周只改 `weekStart` 污染邻周 | **`switchToWeek`** 存旧载新 |
| 11 | 翻邻周时 `convergeWeekFocus` 把日重点写入空周 | **仅当日 ∈ 打开周** 才收敛（存盘层） |
| 11b | **存盘有门闩、表单 input 仍无条件互写 weekFocus** | `bindFormEvents` 同样要 `dayBelongsToOpenWeek()` |
| 12 | 跨周「开启新的一天」克隆上周主题日 | 新周无档 → **空 `days`** |
| 13 | 空日跨周把旧 focus 写入新周 | 跨周空日不写旧 focus |
| 14 | 合并 MIT 清掉目标日完成态 | 同文案完成勾选尽量保留 |
| 15 | 多处自动 mock / 长 placeholder | 仅「示例」；其余空白或短标签 |
| 16 | 示例像产品需求自述 | 中性里程碑文案 |
| 17 | 示例周不完整；清空本日伤周 / 刷新复活 | 示例填满日+周；清空只清日并立即写盘 |
| 18 | 审查滚入：明日有字就算通过 | 须**全部未完成**出现在明日清单 |
| 19 | 预览/MD/TXT 三套字段 | 共用 **`buildPlanView`** |
| 30 | 用难词（深工、身体与恢复、关机仪式堆满） | **白话**：专注工作、运动与休息、睡觉仪式任选 |
| 31 | 运动/复盘可跳过不写理由 | **必回应**；no 要 `skipReason` |
| 32 | 具体安排/边界/反思用大 textarea，MD 一条条但编辑糊 | **事项 = 条目列表**（+一项、Enter、粘贴拆行） |
| 33 | 清空强制 `todayISO()` 误清今天档 | **方案 A：清空当前查看日**，日期不变 |
| 34 | 日∉打开周仍用打开周 `days` 当「今天主题」 | weekStrip **仅日∈周**；MD 与预览对齐 |
| 35 | 切日/清空未取消 debounce 保存 | **`cancelScheduledSave()`** 再切/清 |
| 36 | 睡觉仪式仍强制「两项全勾才总确认」 | 总确认「可以安心睡了」**独立** |

### B. UI / 交互

| # | 错误 | 正确做法 |
|---|---|---|
| 20 | 星期只读或改星期日期不变 | 专用监听；强制同步日期控件 |
| 21 | 下拉选项色点图标 | 选项纯文字；触发器保留箭头 |
| 22 | fixed 菜单滚动悬空 | 页面滚动/resize 关闭（菜单内滚除外） |
| 23 | 手机整页横滑、像双栏侧栏 | 窄屏单栏；表内横滑；`minmax(0,1fr)` |
| 24 | 紫渐变玻璃光晕 AI 风 | 纸色+墨色、小圆角、无光晕 |
| 25 | 主题只跟系统 | 按钮：跟随/浅/深 + 持久化 |
| 26 | 按钮重复（→明日×3）、教学占首屏 | 主路径短；可选折叠；手机底栏仅移动端 |
| 37 | 条目列表每个字段抄一套 render | 公共 `asItemList` / `renderItemList` / `getItemListDefs` |
| 38 | 模块标题/按钮文案像问卷或 AI | 短句、口语、必填用 `req-tag` |

### C. 工程 / 协作

| # | 错误 | 正确做法 |
|---|---|---|
| 27 | 改字段只改预览 | 同步 `buildPlanView` 消费方、审查、示例、文档 |
| 28 | PowerShell 用 bash heredoc 提交 | Windows 用多 `-m` |
| 29 | 堆功能像演示站 | 先闭环诚实 + 默认空白 |
| 39 | 审查只写报告不修 / 修了不更新 skill | 用户确认方案后改代码；**同步沉淀与 skill** |

## 三、用户原话 → 含义（防误读）

| 用户说法 | 含义 |
|---|---|
| 不是输入用户信息 | 不要姓名身份 |
| 默认三件可到六件 | MIT UI 3–6 |
| 全部 mock 不要 / 只有填入示例要 | 严格 mock 边界 |
| 这两个 mock（产品自述句） | 改示例文案 |
| 清空本日不能清周计划 | 清空作用域=日 |
| 下拉现代化→只要文字→图标要 | 列表无图案，触发器有箭头 |
| 夜间跟随系统→要按钮 | 显式主题切换 |
| 太 AI 化 | 去紫渐变玻璃 |
| 下拉跟着滚动条动 | fixed 未关/未跟触发器 |
| 全部都做（滚入+周计划） | 两功能+语义诚实 |
| 从用户体验出发，是否有多余 | 砍重复入口与教学噪音 |
| 整个使用要易懂 | 白话文案；少黑话 |
| 身体与恢复 / 换个词语 / 运动与休息 | 模块名=运动与休息 |
| 提醒自己的不要（起身） | 去掉 standup 起身提醒 |
| 晨间仪式必须 | 审查强制 ≥1 |
| 模块 5/7 必须回应 | 运动/复盘 yes/no + 理由 |
| 具体安排可以添加的一条一个 | `movementPlans` 条目列表 |
| 全部都做（条目列表） | 边界+感恩+明天+反思+周成功/回顾都列表 |
| 进步/做得好/改进 预览与 MD 一条一项 | 反思也列表；导出缩进列表 |
| 关机仪式你觉得需要吗 → A 精简 | 轻量收工；后升级为睡觉仪式 |
| 固定睡前时间改为睡觉时间 | 标签=睡觉时间，字段仍 bedtime |
| 收工与睡觉变为睡觉仪式 + 冥想正念… | `SLEEP_RITUAL` 任选多项 |
| 清空后默认日期为当天 | 曾实现；**后被方案 A 取代** |
| 审查代码和业务逻辑 → a | 修 P0；清空=当前查看日 |
| 根据对话优化 skill 和提示词 | 更新本文件 + skills + prompts |
| git 提交中文 | 中文 commit |

## 五、回归优先级

1. 日闭环诚实（滚入 ≠ 开新日；分档）  
2. 改星期 / 翻周不串档、**表单+存盘**均不污染 focus  
3. 跨周开新日主题日为空  
4. mock 仅示例；**清空=当前查看日**、留周、不误清其它天  
5. 审查三态；运动/复盘/晨间必做规则  
6. 条目列表：预览/MD/TXT 一条一项  
7. 睡觉仪式任选 + 睡觉时间；总确认独立  
8. 下拉滚动、手机布局、无 AI 风

## 七、字段变更同步清单（防漏）

1. `emptyState` / `sampleState` / `normalizeState`  
2. 表单 HTML + 绑定  
3. **`buildPlanView`**  
4. `renderPreview` / `toMarkdown` / `toPlainText`  
5. `auditPlan`  
6. 清空 / 开新日重置范围  
7. 相关 docs + 若行为变了更新 skill
