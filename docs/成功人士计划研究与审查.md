# 成功人士日计划：研究归纳与模板审查

> 目的：从公开方法论中抽取**可移植结构**，对照 `daily-plan-template.html` 做缺口审查与迭代记录。  
> 原则：不抄某人完整作息表，只保留跨角色通用模块。

## 1. 主要参考来源（结构级）

| 来源 / 取向 | 核心做法 | 可移植模块 |
|---|---|---|
| **Ivy Lee Method**（James Clear 等总结） | 每晚写次日最多 6 件；排序；只做第一件直到完成 | MIT 上限 6；默认展示 3；明日清单 |
| **Benjamin Franklin** | 晨间「今日做什么有益」、分时段、晚间自省 | 今日意图；时间块；复盘问句 |
| **Cal Newport · Time Blocking** | 每晚 10–20 分钟排满次日时间块 | 时间块表；典型日骨架 |
| **Deep Work** | 保护无干扰深工；浅工作隔离 | 类型：深度 / 浅工作 |
| **高产者通信习惯** | 邮件/消息 1～2 次集中处理 | 类型：通信 |
| **Miracle Morning (SAVERS)** | 静默/肯定/想象/运动/阅读/书写 | 晨间勾选 |
| **Eat That Frog（Brian Tracy）** | 最难/最重要的事先做 | MIT #1 + 审查「先吃青蛙」 |
| **Tim Ferriss** | Not-to-do list；批处理 | 非待办；通信窗 |
| **GTD（David Allen）** | Capture：先捕获再处理 | 杂念卸载 |
| **Eisenhower** | 重要优先于紧急 | MIT 排序 + 非待办 |
| **精力管理 + Shutdown ritual** | 能量自评；明确结束工作 | 能量 1–5；健康；关机勾选 |
| **主题日 / 主题时段** | 同类事务批量 | 今日主题词 |

## 2. 共性提炼（制定日计划时真正反复出现的）

1. **优先序**：先有限清单，再排时间（Ivy Lee → Time Block）。  
2. **深度保护**：高峰时段给最重要结果，不被消息切割。  
3. **批处理与非待办**：减少切换与假紧急。  
4. **捕获与闭环**：杂念先记下；晚间复盘并预写下一日。  
5. **身体与能量**：睡眠/运动/能量决定深度块长度。  
6. **意图对齐**：日计划服务周/季，而不是随机待办堆。

### 仍刻意不做（避免臃肿）

- 完整艾森豪威尔四象限 UI  
- 番茄钟计时器 / 日历同步  
- AI 自动生成计划  
- 多用户云同步  
- 抄写某名人「5:00 起床」时间表当默认值  

## 4. 推荐用户使用顺序

1. **对齐**：日期 · 能量 · 主题 · 意图 · 周/季 · MIT  
2. **锚定**：填骨架或自建时间块；MIT #1 进上午深度块；设通信窗  
3. **护栏**：非待办 · 健康 · 杂念卸载  
4. **审查**：看「计划审查」未通过项，改到可接受  
5. **闭环**（日终）：复盘 · 明日 MIT · 关机  

## 5. 计划审查项说明

| 审查项 | 依据 |
|---|---|
| 至少 1 件 MIT | Ivy Lee / MIT |
| 主题或意图 | 富兰克林晨问 / 主题日 |
| 有深度块 | Deep Work |
| 上午深度 / 先吃青蛙 | Eat That Frog + 精力高峰 |
| 通信窗 | 高产者邮件习惯 / Ferriss 批处理 |
| 非待办 | Ferriss Not-to-do |
| 周/季对齐 | 目标对齐 |
| 能量 vs 深度负荷 | 精力管理 |
| 明日 MIT | Ivy Lee 晚间步骤 |
| 关机仪式 | Newport Shutdown |
