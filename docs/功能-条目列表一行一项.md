# 功能：事项类输入统一为「一行一项」

## 背景

「运动具体安排」改成可逐条添加后，边界、感恩、明天 Top 3、周成功标准等仍是 textarea + 换行，编辑与 MD 展示不一致。

## 设计原则

| 类型 | 控件 |
|---|---|
| 可数事项 | 条目列表（一行一项，`+ 一项`，可删） |
| 自由反思 | 仍用 textarea（进步 / 做得好 / 可改进 / 跳过理由） |
| 单值 | 单行 input |

## 已改为条目列表的字段

| 字段 | 默认槽位 | 上限 |
|---|---|---|
| 运动具体安排 `health.movementPlans` | 1 | 无 |
| 坚决不做 `notToDo` | 1 | 无 |
| 以后再做 `eisenhower.defer` | 1 | 无 |
| 请别人做 `eisenhower.delegate` | 1 | 无 |
| 临时想法 `capture` | 1 | 无 |
| 感恩 `review.gratitude` | 3 | 无 |
| 明天 Top 3 `review.tomorrowMit` | 1 | 6（与 MIT 一致） |
| 进步 `review.done` | 1 | 无 |
| 做得好 `review.learned` | 1 | 无 |
| 可改进 `review.gap` | 1 | 无 |
| 周成功标准 `week.success` | 1 | 无 |
| 本周回顾 `week.wins` | 1 | 无 |

**仍为段落（textarea）**：不运动 / 不复盘的理由。

## 交互

- **+ 一项** 追加空行  
- **×** 删除（不少于 min 条）  
- **Enter** 在当前项后插入新空行  
- **粘贴多行** 自动拆成多条  
- 预览 / MD / TXT / 审查按列表输出  

## 数据兼容

- 存盘以 `string[]` 为主  
- 读旧档：若是换行字符串，`asItemList` / `linesOf` 自动拆行  
- 运动仍同步拼出 `health.movement` 兼容旧逻辑  

## 实现

`daily-plan-template.html`：

- `asItemList` / `filledItems` / `joinItems`  
- `getItemListDefs` + `renderItemList` / `addItemListRow` / `renderAllItemLists` / `bindItemListAdds`  
- `normalizeState` / `sampleState` / `emptyState` 同步  
- `captureToDefer`、`rollIncompleteMitsToTomorrow`、`startNewDay` 改为吃数组  
- `buildPlanView`、审计、周预览与周 MD 同步

## 附：清空本日与日期

- 「清空」后日期固定为 `todayISO()`（当天实时日期），不再沿用清空前选中的其它天。  
- 「清除全部本地数据」本身走 `emptyState()`，日期本来就是当天。
