# 提示词：审查产品业务与代码

---

审查 `daily-plan-template.html` 的**业务逻辑**与**代码逻辑**。  
对照：`AGENTS.md`、`docs/对话沉淀-需求与教训.md`、`.grok/skills/make-plan-lessons/SKILL.md`、近期 `docs/审查-*.md`。

## 业务必查

### 主流程与闭环

- 主路径：对齐 → MIT → 时间块 → 护栏 → 晨间/运动 → 复盘/`→ 明日`/睡觉仪式 → 开启新的一天  
- 「→ 明日」与「开启新的一天」语义是否诚实  
- 存档是否按日/周；`switchToDate` / `switchToWeek` 是否先 `cancelScheduledSave` 再存后载  

### 串档与 focus（P0 级）

- `convergeWeekFocus` 是否仅在日∈打开周  
- **`bindFormEvents` 里 weekFocus 是否同样有门闩**（历史 bug：存盘对、表单错）  
- 跨周开新日是否克隆上周 `days`  
- 预览 weekStrip 是否在日∉周时仍取邻周主题  

### 清空 / mock

- 清空 = **当前查看日**、日期不变、留周、不误清其它天  
- mock 是否仅示例入口；示例日+周完整无自述  

### 必做与审查

- 晨间 / 运动 willMove / 复盘 willReview 规则  
- 睡觉仪式：任选 + 睡觉时间 + 总确认独立（非全勾联动）  
- 审查三态与滚入全覆盖  
- 条目列表是否进审查/预览  

## 代码必查

- `buildPlanView` 是否仍是预览/导出唯一数据源  
- MD/TXT 是否与预览同结构（列表、主题条、落款用计划日）  
- 条目列表公共组件 vs 复制粘贴 DOM  
- `normalizeState` 是否兼容旧字符串/旧 windDown  
- 表单按钮 ID 是否有死链  
- 日期星期绑定、下拉滚动关闭、手机溢出  
- XSS：预览是否 escapeHtml  

## 输出

1. 总评与分项  
2. **必须修 / 应修 / 可延后**（标 P0/P1/P2）  
3. 若用户选方案（如清空 A/B）或说「需要/全部修复」：按优先级改代码 → 跑回归 → 更新 `docs/` 与 skill（行为变了时）
