# 提示词：审查产品业务与代码

---

审查 `daily-plan-template.html` 的**业务逻辑**与**代码逻辑**。

## 业务必查

- 主流程：对齐 → MIT → 时间块 → 护栏 → 勾选 → 复盘/`→ 明日`/关机 → 开启新的一天  
- 「→ 明日」与「开启新的一天」语义是否诚实  
- 存档是否按日/周；`switchToDate` / `switchToWeek` 是否先存后载  
- `convergeWeekFocus` 是否仅在日∈打开周  
- 跨周开新日是否克隆上周 `days`  
- 清空本日 vs 清除全部  
- 审查三态与滚入全覆盖  
- 关机双向  
- mock 是否仅示例入口  

## 代码必查

- `buildPlanView` 是否仍是预览/导出唯一数据源  
- 表单按钮 ID 是否有死链（精简 UI 后）  
- 日期星期绑定、下拉滚动关闭  
- 手机溢出  
- XSS：预览是否 escapeHtml  

## 输出

1. 总评与分项评分  
2. 必须修 / 应修 / 可延后  
3. 若用户说「需要/全部修复」，按优先级改代码并回归
